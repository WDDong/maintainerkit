# Security Policy

MaintainerKit may inspect issue text, pull request metadata, changed file paths, and release metadata. It should not collect secrets or persist repository content outside the GitHub Action runtime.

## Reporting a vulnerability

Please report security issues privately by opening a GitHub security advisory if available, or by contacting the maintainer directly.

Do not open a public issue for a vulnerability until a maintainer confirms disclosure is appropriate.

## Supported versions

MaintainerKit is pre-1.0. Security fixes target the latest release line.

## Maintainer expectations

Security-related labels and comments are advisory. Maintainers should verify any report before disclosure, patching, or public discussion.
